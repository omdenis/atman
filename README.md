# atman

*An operating system for the self — life as a system you tend, not a future you chase.*

**atman** is a personal-development methodology realized as a plain-text, git-versioned
Obsidian vault. It replaces thinking in *states*, *projects*, and *someday-futures* with one
discipline: **run the system, improve it by +1 each block, and let knowing arrive through
experience — not theory.**

Everything you decide, observe, or learn is captured *immediately* to a file. Nothing lives
only in your head or in an AI's memory. The vault is the source of truth; the agents are
just hands.

## The shift in mindset
- **From states → to a system.** Stop managing moods and milestones; tend one living system.
- **From outcomes → to +1.** Each work-block leaves the system one increment better —
  compounding beats chasing.
- **From living-in-the-future → to the present log.** The plan is a *projection* of what's
  true now, rebuildable at any moment.
- **From borrowed theory → to experiential knowing.** Principles are earned from logged
  experience, not copied from books.

## Core principles
- **Event sourcing.** An append-only log of events and decisions is the single source of
  truth; every dashboard, plan, and decision-matrix is a *projection* rebuilt from it. Events
  are never edited — a correction is a new event that `supersedes` the old one.
- **Immediate capture.** *"Not written to a file = does not exist."* Agent memory (SQLite,
  MEMORY.md) is a **cache** that can be rebuilt from the vault.
- **Calibrated decisions.** Every decision carries an `expected_outcome`, a `confidence`
  (1–10), and a `review_at` date — a forecast with feedback, not just a choice.
- **Decisions carry their context.** A significant artifact is the *result* of a recorded
  decision (the problem, the alternatives rejected, the why) and links back to it —
  provenance is built into the model, so you can always trace where the legs grow from.
- **Self-developing.** The methodology develops itself inside `systems/atman/` — literally
  the machinery that builds the machinery: a system that improves the rules by which it
  improves.
- **Ritual at the threshold (sandhya).** Each work-block is opened and closed deliberately —
  read the latest handoff, frame, act; then synthesize the block into a review and write a
  new handoff that ends mid-stride (a "Hemingway bridge"), so the next session starts
  cold-free.

## Why "atman"
*Ātman* (Sanskrit) is the higher Self — the unchanging witness behind the bustle of daily
life. The name doesn't describe the contents (those age); it names *who lives here*. Where an
*avatar* is the higher descending into a form — a projection outward — **atman is the source
those projections come from.** The vault is that source.
